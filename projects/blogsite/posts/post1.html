Building My First Website From Scratch

Creating a website from scratch taught me more than any tutorial ever could. Watching someone else build a project is one thing, but starting with a completely blank HTML file forces you to actually understand how everything fits together. There’s no structure handed to you, no pre-made layout — just an empty file and the responsibility to turn it into something functional.

At first, that simplicity was overwhelming. Even basic decisions like where to start or how to structure the page felt uncertain. But once I broke it down into small steps — starting with the layout, then adding content, then styling — the process became much more manageable. It stopped feeling like “building a website” and started feeling like solving a series of smaller problems.

I focused heavily on getting the fundamentals right. That meant setting up a clear HTML structure first, before touching any styling. I wanted the layout to make sense logically, not just visually. From there, I gradually added CSS to bring the design to life, paying attention to spacing, alignment, and readability rather than trying to over-design anything.

One of the biggest learning curves was CSS layout. Understanding how elements behave in the document flow, and how properties like flexbox affect positioning, changed how I think about design. Small adjustments in margin, padding, and container width had a much bigger impact than I expected. I also started to understand how important consistency is — once spacing or typography is off in one place, it affects the entire feel of the site.

Responsive design was another challenge. Making the site look good on different screen sizes forced me to think more carefully about structure instead of relying on fixed layouts. It made me realise that good design isn’t just about how something looks on your screen, but how it adapts to different environments.

I also became more aware of how messy code can get without planning. Early on, I was just adding styles as needed, but as the project grew, I had to go back and reorganise things to make it easier to manage. That experience alone taught me the value of writing clean, structured code from the beginning.

The final result isn’t perfect, and I can already see things I would improve if I rebuilt it today. But that’s exactly why this project matters. It represents the point where everything stopped being purely theoretical and started becoming practical. It’s the first real step in building confidence as a developer, and a foundation for every project that comes after it.
